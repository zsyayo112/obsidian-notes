# 📌 函数作为值 & 回调函数
> Topic: [[JS 异步基础]] · Tier: 1 · 难度: ⭐
> 💡 今天所有困惑的「根」——`.then` / `setTimeout` / `onClick` / `setState` 全靠这一块。

## 📖 0. 术语扫盲
| 英文 | 中文 | 一句话解释 |
|---|---|---|
| First-class function | 一等函数 | 函数是个「值」，能存进变量、能当参数传 |
| Callback | 回调函数 | 把函数交出去，让别人在合适时机替你调用 |
| Anonymous function | 匿名函数 | 不起名字、直接写出来的函数（`function(){}` / `()=>{}`） |

## 📝 1. 实例代码
```javascript
// 代码读法说明：
// - 前半：f 和 f() 的区别（函数本身 vs 立刻执行）
// - 后半：把函数交给别人，别人替你调用 + 替你塞参数（= 回调）

// ① 函数是个值，能存能传
function sayHi() { console.log("hi"); }
sayHi;     // 函数本身（一张菜谱，还没做菜）—— 不加 ()
sayHi();   // 立刻执行（照菜谱做菜）—— 加了 ()
const f = sayHi;   // 把函数本身存给 f，没有执行
f();               // 用 f 调用，打印 hi

// ② 回调：把函数交出去，接收方替你调用、替你塞值
function 执行并塞值(cb) {
  cb("汉堡");            // 接收方调用你的函数，并塞一个值进去
}
function 我拿到啥(东西) {   // 声明参数「东西」，但我自己不调用它
  console.log("我拿到了：" + 东西);
}
执行并塞值(我拿到啥);   // 传函数本身（不加 ()）→ 输出：我拿到了：汉堡
```
⚠️ 易错点：
1. **传函数本身 vs 立刻执行**：`setTimeout(叫号, 1000)` 不加 `()`；加了 `()` 变成「现在立刻执行、把返回值传进去」。
2. **回调的参数是接收方塞的**：你只声明参数名，「什么时候调用 + 塞什么值」都由接收方决定（`.then(食物=>)` 的食物、`setList(prev=>)` 的 prev 都是别人塞的）。

## 🎯 2. 使用场景
1. **SaaS 场景**：事件监听 `button.addEventListener('click', 回调)`——用户点击时浏览器替你调用回调。
2. **Fintech 场景**：异步交易完成后的处理 `支付().then(结果 => 更新账本)`——支付完成时 Promise 替你调用。
3. **E-commerce 场景**：React 列表渲染 `items.map(item => <Card />)`——map 对每个元素替你调用你传的函数。

## 🔄 3. 可替代技术
| 替代方案 | 优势 | 劣势 | 何时用 |
|---|---|---|---|
| 具名函数当回调 | 可复用、可调试（栈里有名字） | 要多起个名字 | 逻辑复杂、要复用时 |
| 匿名函数/箭头函数当回调 | 简洁、就地写 | 只用一次、栈里没名字 | 简单逻辑、只用一次 |

**选型决策树**：
- 如果回调逻辑长/要复用 → 具名函数
- 如果就一两行、只用一次 → 箭头函数就地写

## 💼 4. 面试题
### 题 1（🟢 基础）
**Q**: `setTimeout(fn, 1000)` 里的 `fn` 为什么不加括号？加了会怎样？
**核心 points**: 不加 = 传函数本身让 setTimeout 等会儿调用；加了 = 立刻执行、把返回值(通常 undefined)传进去，且不会延迟。
**追问方向**:
1. 如果一定要传参怎么办？（用箭头函数包一层 `() => fn(arg)`）
2. 回调里的 `this` 会怎样？（可能丢失，见 [[06-this 五种绑定]]）
**加分回答**: 「函数是一等公民」是 JS 的核心特性，回调、高阶函数、函数式编程都建立在此之上。

## 🧪 5. 小测验
> 点击展开答案前先思考：下面输出什么？
```javascript
function greet(name) { console.log("Hi " + name); }
setTimeout(greet, 0);          // A
setTimeout(() => greet("Hal"), 0);  // B
```
<details>
<summary>展开答案</summary>

- A → `Hi undefined`：setTimeout 调用 greet 时没塞参数，name 是 undefined。
- B → `Hi Hal`：箭头函数里我们**主动**调用 `greet("Hal")` 并传了值。
</details>

## 🌟 Senior 思维彩蛋
把方法当回调传出去（`setTimeout(user.greet, ...)`）时，`this` 会丢——这是生产事故常见原因。Senior 会条件反射地检查「这个回调传出去后，它依赖的 this/闭包还在吗」。根治方式是箭头函数或 bind。
