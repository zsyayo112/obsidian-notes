# 🌳 二叉树 Hot 100 通关复习卡

> 一句话总纲：**二叉树 90% 的题 = 递归。你要练的不是背 18 道解法，而是「看一道题，先判它属于哪一类」。**
> 18 题全通（A/B/C/D/E 五类），含 1 道 Hard（124）。本卡是单一复习入口。

---

## 一、五大题型地图（核心：先归类，再套路）

| 类 | 名称 | 一句话特征 | 用什么 | 题目 |
|---|---|---|---|---|
| **A** 🟢 | 遍历收集型 | 走一遍，把结果塞进外部容器 | 遍历 + 数组/队列 | 94、102、199 |
| **B** 🟢🟡 | 子树拼装型 | **函数 return 的 = 要的答案**，左右一拼就出 | 递归三部曲，纯 return | 100、104、226、101 |
| **C** 🔴 | 上报+全局型 | **return 零件 ≠ 最终答案**，答案另存全局变量 | 三部曲 + 全局变量 | 110、543、124 |
| **D** 🟡 | BST 有序型 | 利用「中序遍历 = 升序」 | 中序遍历 | 230、98、108 |
| **E** 🔴 | 路径/构造/LCA | 各有专门套路 | 一题一招 | 236、105、437、114 |

**判类口诀（做新题第一步）**：
> 问自己：「我要的东西，对子树是**同一种东西**、能由左右子树拼出来吗？」
> - 能 → **B 类**（return 即答案）
> - 不能、但需要按顺序收集一串 → **A 类**（外部容器）
> - 不能、函数能返回有用的「子树零件」但它 ≠ 最终答案 → **C 类**（全局变量存答案）
> - 是 BST 且和「有序」有关 → **D 类**（中序）

## 二、万能模板：递归三部曲

写**任何**二叉树递归，按顺序填空：

```
① 这个函数返回什么？   ← 先用一句话写死「返回 XX」（最重要的一步）
② 什么时候停？         ← if node is None: return 某个值
③ 这一层做什么？       ← 用左右孩子的结果，拼出我自己的结果
```

> 90% 的卡壳，都是第①步没定清楚。**能用一句话说清「函数返回什么」，剩下两步自然填得出。**

## 三、两种解题思维（A/B 类的分水岭）

| | 遍历思维（A 类） | 分解问题思维（B 类，主力） |
|---|---|---|
| 答案存哪 | 外部变量 | 函数 return 值 |
| 信息方向 | 自上而下（参数往下传） | 自下而上（返回值往上冒） |
| 递归函数返回 | `void`（不返回） | 返回子问题的解 |
| 像哪种遍历 | 前序（进门即处理） | 后序（左右算完才处理） |

**B 类统一骨架**：
```python
left = 递归(node.left)        # 先要到左子树答案
right = 递归(node.right)      # 先要到右子树答案
return 用 left, right, node 组装出我的答案
```

## 四、私有 vs 公有判断法（决定「传参数」还是「全局变量」）

> **唯一标准：换一条递归路径，这个值会变吗？**
> - **会变 → 私有（参数传下去）**：路径和、深度、到根距离、当前路径
> - **不变（整个问题只有一份答案，要汇总所有路）→ 公有（全局变量）**：有没有出现过 X、总共几个、全局最大值、计数器

迁移性极强 —— 回溯 / DFS / 图遍历同一套心法。

## 五、递归栈模型（理解「深度为什么自底向上冒」）

递归 = 「递」下去 + 「归」上来：

| 阶段 | 栈动作 | 算 1+max 吗 | 方向 |
|---|---|---|---|
| **递** | 压栈，函数卡在 `递归(left)` 等返回 | ❌ 只往下钻 | 根 → 叶 |
| **触底** | `None` 命中 Base case，return 0 | ❌ 直接返回 | 最深处 |
| **归** | 弹栈，算 `1+max` 并 return 给上层 | ✅ 这里才算 | 叶 → 根 |

- `return` 的本质 = **弹栈时把成果交给「挂起等待我的上一层」**。少了 return，上层永远等不到值 → 崩。
- **接收和返回必须配对**：写了 `x = 递归(...)` 去接，就必须 return；否则改用「原地修改、不接收返回值」的 void 写法。

## 六、关键单点知识

### `1 + max(left, right)`（树高/深度命根子）
- `max`：往下只能选**一个孩子方向**走，挑更深的那条腿。
- `+1`：算上「我自己这一层 / 我到孩子的那条边」。
- 整体：把「孩子的深度」加工成「我的深度」，return 给父亲。

### C 类灵魂：上报值 ≠ 全局答案
以 543 直径为例：
- **return 给父亲**（单边）：`1 + max(左, 右)` —— 父亲只能从我伸一条腿下来。
- **更新全局**（拐弯）：`左 + 右` —— 路径在我这拐弯，左右两腿都算。

### BST 性质（D 类命门）
- **中序遍历（左→根→右）= 升序序列**。
- 求「第 k 小」= 中序第 k 个；「第 k 大」= 反中序（右→根→左）第 k 个。
- 「处理逻辑」放递归调用的**前/中/后**，决定前序/中序/后序 —— 位置 = 顺序 = 对错。
- 验证 BST 不能只比父子，要用「中序严格升序」（祖先约束才不会漏）。

### 切片 / 空判断（105 易错）
- Python 切片 `arr[start:end]` 的 `end` 是**终止下标**，不是元素个数。「取到末尾」→ 右边界留空 `arr[start:]`。
- 切片永远返回**列表**（可能是 `[]`），不会是 `None` → Base case 用 `if not preorder`，不能用 `is None`。
- `[] is None` 是 `False`；`not []` 是 `True`。

## 八、⚠️ 二刷优化待办（暴力解已 AC，二刷升级）

- [ ] **105 构造树**：当前 `inorder.index()` + 切片是 **O(n²)**。优化：用 `dict` 存「值→下标」(O(1) 查找) + 索引边界代替切片 → **O(n)**。
- [ ] **437 路径总和 III**：当前双层递归是 **O(n²)**。优化：**前缀和 + 哈希表**一遍 DFS → **O(n)**。核心公式：站在前缀和 `s_i` 的节点，「以我结尾、和=target 的路径数」= 「前面前缀和等于 `s_i - target` 的出现次数」。树上要用**回溯**把离开节点时的前缀和计数撤销。
- [ ] **124 / 236**：能口头讲清复杂度 + trade-off（面试加分）。

## 九、🧪 Spaced Repetition 复习卡片（插件格式）

「换一条递归路径这个值会变吗」判断的是什么？::私有(参数) vs 公有(全局)。会变=私有往下传；不变、整树一个答案=公有全局变量。

`1 + max(left, right)` 里 `+1` 和 `max` 各是什么意思？::`max`=往下只能选一个孩子方向走挑更深的；`+1`=算上我自己这一层/我到孩子的那条边。

C 类（上报+全局型）和 B 类（子树拼装型）的本质区别？::B 类 return 的就是答案；C 类 return 的是「拼答案的零件」，答案要另设全局变量存。

BST 求「第 k 小」用什么遍历？「第 k 大」呢？::第k小=中序(左根右)第k个；第k大=反中序(右根左)第k个。

543 直径：return 给父亲用什么公式？更新全局用什么？为什么不同？::return=`1+max(左,右)`(当腿只能选一边)；全局=`左+右`(拐弯两腿都算)。

124 比 543 多了什么关键操作？::`max(0, gain(child))` —— 孩子贡献为负就砍掉(当0)，因为路径可以在变负前停下。

105 构造树：`i = inorder.index(根)` 这个 i 同时是什么？::既是「中序里根的下标」，又是「左子树的节点个数」，靠它同时切前序和中序。

什么时候递归函数可以不写 return？::当它「原地修改传进来的对象」(改指针)、且不用变量接收返回值时(void 模式，如 114 展平)。算新值要交出去的必须 return。

## 十、🌟 面试加分句（Senior 视角）

- **递归的空间复杂度不是 O(1)**，是 O(树高) —— 因为借用了系统调用栈，每层压一帧。处理深度不可控的用户数据时，优先迭代版防爆栈。
- **序列化树用前序**：反序列化时前序第一个天然是根；只给中序无法唯一还原一棵树。
- **「何时该优化」**：先写清晰版(105 切片、437 暴力)能 AC，再说「我知道这是 O(n²)，可用哈希表/前缀和降到 O(n)」—— 这句就是 mid 和 senior 的差距。
