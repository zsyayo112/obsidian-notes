# 🗺️ Agent 开发 MOC

> [!abstract] 一句话总纲
> **Agent = 大模型 + 一个 while 循环 + 一组工具函数。**
> 模型只会吐文本，它"调用工具"只是吐出一段表达意图的结构化文本；真正执行的是你的代码，执行结果再塞回对话历史，重新问模型。循环往复，直到模型说"我搞定了"。

这套笔记来自一次"手把手、边学边做"的实践：不用任何框架，从裸 SDK 写到一个能跑的命令行编程助手 `codeagent`。

## 🧠 贯穿全程的核心原理

这几条不属于某一课，而是从头到尾反复出现——Obsidian 图谱里它们连着每一课：

- [[硬约束vs软约束]] — 提示词能哄的地方都能被无视，安全靠代码
- [[错误即信息]] — 工具失败不要崩，把错误当结果喂回模型
- [[消息配对约束]] — messages 不能随便切，tool_calls 和结果必须成对
- [[幻觉 模型看不见就编]] — 模型的世界只有你发给它的 messages
- [[路径困惑 一个反复出现的坑]] — 同一个坑在 L3/L4/L5/L6/L7 出现了五次
- [[子agent模式要匹配任务形状]] — 调查员（大进小出）vs 建造者（小进大出），用错是负收益

## 🎓 毕业时你拥有了什么

一个不到 300 行、不依赖任何框架、你理解每一行的编程 agent（`agent/` 包）、一个把它长成真实仓库 issue agent 的毕业设计（[[RepoPilot MOC|RepoPilot]]，已开源），以及一套"读代码就知道会发生什么"的确定性思维**为什么在 agent 里失效**的第一手经验。
