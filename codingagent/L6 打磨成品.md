# L6 · 打磨成品

> [!abstract] 核心
> 把散落各课的代码收进规整的 `agent/` 包，加上**流式输出**，`pip install -e .` 成一个真能用的 `codeagent` 命令。玩具脚本 → 项目的关键：**改一处不会碰坏另一处**。

## 包结构（职责分离）

```
agent/
├── config.py       环境、client、常量（唯一配置来源）
├── tools.py        工具实现 + schema + 执行（纯逻辑，不含权限）
├── permissions.py  权限闸门
├── context.py      sanitize 守卫 + 压缩 + 截断
├── loop.py         agent 循环 + 流式输出
└── cli.py          薄薄的 REPL，只管读输入和分派
```

## 唯一的新技术点：流式拼装

流式响应被切成很多 chunk：
- **文本片段**在 `delta.content`，一段段来，直接打印 → "打字机效果"。
- **工具调用**在 `delta.tool_calls`，而且**参数是逐字符流式拼出来的**，必须按 `index` 把碎片攒齐才得到完整调用。
- 要 token 用量得加 `stream_options={"include_usage": True}`，且收尾 chunk 的 `choices` 是空的，要先判断。

这段拼装逻辑就是流式 agent 比非流式多出来的全部复杂度。

> [!important] 带走
> **"测试通过"只证明它测的东西对了，不证明没测的东西也对。** 隔离性、清理、无用 import 都在"测试通过"的盲区里。**验收 agent 的产物，永远要自己再看一眼真实产物（ls / cat）。**

修 bug 时它做对了一件事：用**依赖注入**（`TODO_FILE_PATH` 环境变量）改造生产代码的可测试性，而非将就改测试。但仍留两个尾巴：`import tempfile` 又没用上、`test_todos.json` 跑完不清理。
