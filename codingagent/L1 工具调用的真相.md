# L1 · 工具调用的真相

> [!abstract] 核心
> **模型没有能力读文件、连网、碰硬盘。它"调用工具"只是吐出一段结构化文本表达意图（函数名 + 参数）；真正执行的是你的代码。** 这是整个 agent 最关键的一课。

## 四幕慢放

1. **问模型**（附上 TOOLS 说明书）→ 它返回的 `content` 是空的，意图塞在 `tool_calls` 里，`finish_reason = tool_calls`。
2. **此刻文件读了吗？没有。** 一个字节都没读。模型只吐了函数名和参数（`arguments` 是**字符串**不是 dict，所以偶尔会是语法错误的 JSON）。
3. **你的代码去执行**：`result = read_file(**args)` —— 整个 agent 里唯一"与世界交互"的地方。然后把这一来一回 append 进 messages（assistant 那条 + `role="tool"` 那条，用 `tool_call_id` 配对）。
4. **拿新 messages 再问一次** → 模型给出最终答案，`finish_reason = stop`。

## 关键认知

- 工具的 `description` 是**提示词工程**，不是代码注释——写得好不好直接决定 agent 聪不聪明。
- 真正能碰硬盘/网络的只有你的 Python 函数，模型永远够不着。这也是 [[L5 权限与硬约束]] 安全模型的全部基础。
- `tool_call.id` 用来配对结果，因为模型可以**一次要求并行调多个工具**。见 [[消息配对约束]]。
