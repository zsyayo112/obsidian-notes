# L5 · 权限与硬约束

> [!abstract] 核心
> 改动性操作动手前必须停下来问用户。而**能拦住的规则，必须用代码拦，不能靠提示词哄**——这是 agent 安全的分水岭。见 [[硬约束vs软约束]]。

## 三个设计决策

1. **工具分级**：`SAFE_TOOLS = {read_file, list_files}` 自动放行；写文件/执行命令必须确认。判断标准：这个工具会改变世界的状态吗？
2. **确认三选项**：`y` 同意一次 / `a` 本会话全放行 / `n` 拒绝。拒绝时理由作为**工具结果**喂回模型——[[错误即信息]]，**拒绝即对话**。
3. **路径根治**：在 `_resolve` 里剥掉模型误加的前缀。提示词管不住的，代码兜底。

## 分水岭那句话

> [!important]
> 用提示词让模型别做坏事，是所有新手的第一反应，也是所有事故的来源。**能用提示词哄的地方，都是它可以不听的地方。真要拦住的，必须在代码里拦。**

## 亲历的坑

`/trust` 开启后，工具调用行仍显示"(需确认)"标签，但实际没拦——**显示与行为不一致**。根因：标签只看工具类别，没看 `trust_all` 状态。这类 UI bug 不致命，但会摧毁权限系统的可信度。修法：让标签严格反映"这次到底会不会拦"。
