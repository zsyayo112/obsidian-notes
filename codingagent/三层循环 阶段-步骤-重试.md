# 🔄 三层循环：阶段 / 步骤 / 重试

> [!important] 一句话
> **RepoPilot 里有三个 `for`/`while`，长得像但迭代的"单位"完全不同。看清它们，就看清了整个 agent 的骨架。**

## 三层（粒度从大到小）
```
orchestrator  while state != "DONE"     ① 状态循环：一圈 = 一个阶段（基线/计划/执行/验证/复查/报告）
   └─ EXECUTE 调 run_executor
        for _ in range(MAX_TURNS)        ② 干活循环：一圈 = 一个工作步骤（读/搜/改/跑测试）
   └─ PLAN 调 make_plan → json_call
        for _ in range(retries)          ③ 重试循环：一圈 = 一次"把格式答对"的尝试
```

## 三者的区别

| | ①状态循环 | ②干活循环 | ③重试循环 |
|--|--|--|--|
| 迭代单位 | 阶段 | 任务步骤 | 格式尝试 |
| 每圈是 | 换一个阶段 | 往前走一步 | 同一问题重问 |
| 停止条件 | state=DONE | 模型不再要工具 / MAX_TURNS | 拿到合法 JSON / 用完次数 |
| 失败指 | —— | —— | 输出格式不对 |

注意 ②③ 在**不同分支**，并不嵌套：executor 走自己的流式调用，planner/reviewer 才走 json_call。

## 暗线：三者是同一个动作
本质都是**"喂点东西回去 → 再问一次"**（[[错误即信息]]的放大）：状态循环喂回测试对比，干活循环喂回工具结果，重试循环喂回格式错误。
> **agent 的本质，就是"把结果喂回、再问一遍"这个动作，在三种粒度上各转一个圈。**

## 干活循环怎么停
模型不会喊"停"。它这一轮**只回文字、不再要任何工具**，就是"我干完了"的信号 → `if not tool_calls: return`。另一个出口是撞 MAX_TURNS 被**强制掐停**（含义相反：一个是做完，一个是被切断）。
