# 🪓 两大类分层：碰不碰 LLM

> [!important] 一句话
> **把 agent 的文件按"会不会发消息给模型"切成两大类，是理解整个架构最重要的一刀。核心手艺 = 把"不确定性"关进尽量小的范围。**

## 怎么判定
看这个文件里有没有 `get_client()` / `json_call`（通向模型调用的唯一入口）。有 = 碰 LLM，没有 = 不碰。

## 两类的差别

| | 不碰 LLM（身体）| 碰 LLM（大脑）|
|--|--|--|
| 干嘛 | 读写文件、跑 git、跑测试、做判断 | 把话发给模型、拿回模型的话 |
| 确定性 | 每次结果一样 | 每次可能不一样 |
| 要钱/网 | 不要，离线秒出 | 要 key、花 token |
| 能单测 | 能 | 难 |

## RepoPilot 的分法
- **不碰（7）**：config / workspace / policy / tools / adapters / verifier / trace
- **碰（4）**：llm / planner / executor / reviewer
- **只指挥、自己不碰**：orchestrator / cli（调别的模块去碰，自己是纯控制流）

## 为什么重要
把"跟世界打交道"和"跟模型打交道"彻底分开：身体那半确定、可测、离线跑（20 个单测全在这、都不要 key）。**把不确定性锁在少数几个文件里**，是 agent 工程的核心手艺。

## 比喻
模型 = 按次收费、发挥不稳的天才顾问。不碰 LLM 的代码 = 你自己的手脚工具箱（可控免费）；碰 LLM 的代码 = 去敲顾问门的那几扇门。能自己动手就别问顾问，只在需要判断/创造时敲门，且把门集中管理。
