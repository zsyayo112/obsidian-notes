# 🐚 薄壳原则：接口与核心解耦

> [!important] 一句话
> **接口层（CLI）做得尽量薄，只负责输入输出，业务逻辑全在核心（`solve()`）里。这样换界面——CLI / Web / MCP——都不用动核心。**

## cli.py 有多薄
只干三件事：①解析参数 ②搞清 issue 文字从哪来（`--issue`/`--issue-file`/`--issue-gh` 三选一）③交给 `orchestrator.solve()`。真正的活儿一行都不在 CLI。
```python
if args.issue:        issue = args.issue
elif args.issue_file: issue = Path(args.issue_file).read_text()
else:                 issue = fetch_issue(args.issue_gh)   # github.py
code = solve(args.repo, issue, ...)                         # 递进核心，自己不掺和
sys.exit(code)                                              # 退出码 0=成功，供脚本/评测
```

## 为什么薄壳值钱
`solve()` 跟"命令行"毫无耦合。所以：
- 想变 **Web 服务**？写个 HTTP handler 调同一个 `solve()`。
- 想包成 **MCP server**？把 `solve()` 暴露成一个 MCP 工具。
- **换界面不动核心**——这就是解耦的红利。（呼应最初关心的"包成 MCP"，见 [[MCP 是什么]]。）

## detect 子命令
`repo-pilot detect` = 免费试探（不调模型），先看 adapter 认成什么。一个廉价 dry-run。

## 面试
"怎么设计 agent 的对外接口 / 怎么让它既是 CLI 又能当服务" —— 薄壳：接口只做 IO，核心与接口解耦，CLI/Web/MCP 共用同一个 `solve()`。
