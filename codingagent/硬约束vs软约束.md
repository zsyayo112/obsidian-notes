# 🧱 硬约束 vs 软约束

> [!important] 一句话
> **提示词是软约束（模型可以不听），代码是硬约束（模型没得选）。安全和正确性永远建立在硬约束上。**

## 区别

| | 软约束 | 硬约束 |
|--|--------|--------|
| 载体 | system prompt、工具 description | Python 代码里的检查、拦截、剥离 |
| 效力 | 模型"心情好"才听；换模型/调高 temperature 就可能失效 | 模型绕不过去 |
| 用途 | 引导风格、倾向、默认行为 | 安全、数据完整性、绝不能破的规则 |

## 亲历的铁证

在 [[L6 打磨成品]] 里，system prompt **白纸黑字**写着"不要在命令里 cd 到工作目录"，模型**读到了、然后照样 `cd /playground`**。它不是没看见，是看见了不听。参见 [[路径困惑 一个反复出现的坑]]。

真正拦住"嵌套目录灾难"的，从来不是那句提示词，而是 [[L5 权限与硬约束]] 里 `tools._resolve()` 中剥前缀、`is_relative_to` 挡越狱的**代码**。

## 落到代码上的硬约束清单

- 路径 `resolve()` 后校验 `is_relative_to(沙盒)` → 挡越狱
- 无条件剥掉模型误加的 `playground/` 前缀 → 防嵌套
- `SAFE_TOOLS` 集合 + 权限闸门 → 改动性操作强制确认
- 工具执行 `try/except` 永远返回字符串 → 见 [[错误即信息]]

## 带走

> 你的第一反应会是"我加一句提示词让它别这么干"。**能用提示词哄的地方，都是它可以不听的地方。真要拦住的，必须在代码里拦。**
