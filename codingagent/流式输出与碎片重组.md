# 🌊 流式输出与碎片重组

> [!important] 一句话
> **流式让文字一个字往外蹦（体验好），代价是：工具调用被拆成碎片流回来，参数要一个字符一个字符拼、按 index 攒齐，才能还原成一次完整调用。**

## 非流式 vs 流式
- 非流式：等模型把整段生成完，一次性拿到 `message.tool_calls`——简单。
- 流式：模型一小块一小块（chunk）吐，你边收边处理。文字块直接打印（"打字机"效果）；但**工具调用的参数是逐字符流式拼出来的**。

## 碎片重组（executor._stream_once 的核心难点）
```python
tool_buf = {}                         # index -> 正在拼的 tool_call
for chunk in stream:
    delta = chunk.choices[0].delta
    if delta.content:                 # 文字块：立刻打印
        print(delta.content, end="", flush=True)
    if delta.tool_calls:
        for tcd in delta.tool_calls:
            slot = tool_buf.setdefault(tcd.index, {"id":None,"name":"","arguments":""})
            if tcd.function.arguments:
                slot["arguments"] += tcd.function.arguments   # 参数一点点拼起来
# 按 index 排序还原成有序的 tool_calls
tool_calls = [tool_buf[i] for i in sorted(tool_buf)]
```
关键：**按 `index` 分槽**（一次可能并发多个工具调用），每个槽的 `arguments` 字符串一段段追加，流结束才完整。

## 还有一个坑
带 token 用量的**收尾 chunk 里 `choices` 是空的**，要先判 `if not chunk.choices: continue`，否则会崩。

## 面试
"streaming 怎么处理 / 流式和非流式的区别" —— 文字直接打印，工具调用要按 index 攒碎片重组参数；这是流式体验换来的复杂度。
