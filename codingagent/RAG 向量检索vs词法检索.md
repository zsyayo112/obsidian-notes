# 🔎 RAG：向量检索 vs 词法检索

> [!important] 一句话
> **RAG（检索增强生成）= 先从知识库检索相关内容、塞进提示词，让模型基于它回答。检索有两条路：词法（精确匹配 token）和向量（语义相似）。RepoPilot 故意用词法（ripgrep），因为代码要精确符号匹配。**

## RAG 是什么
Retrieval-Augmented Generation：模型不靠记忆答，而是**先检索**相关片段、注入上下文，再生成。解决"知识不在模型里/会过时/会编"的问题。

## 两种检索
| | 词法/关键词（BM25、grep/ripgrep）| 向量/语义（embedding）|
|--|--|--|
| 原理 | 匹配精确的词/token | 把文本变成向量，找最近邻（余弦相似）|
| 强项 | 精确、快、无需 embedding | 抓语义/近义/换个说法也能命中 |
| 弱项 | 换个说法就搜不到 | 可能召回"意思像但不对"的 |
| 典型件 | ripgrep、Elasticsearch | 向量库(FAISS/pgvector/Chroma) + embedding 模型 |
向量检索的常见流程：切块(chunk) → 嵌入(embed) → 存向量库 → 查询也嵌入 → top-k 最近邻 → （可选 rerank）。

## RepoPilot 为什么用词法（ripgrep）
**代码检索要精确符号匹配**：搜 `def __le__`，你要的就是这个 token，不是"语义相似的代码"。所以用 ripgrep（词法）而非向量。这是**有意的选择**，不是没想到向量。

## 什么时候我才会上向量
当查询是**自然语言、且措辞和目标不一样**时——比如"按含义搜文档/知识库"。生产里常用**混合检索**（词法+向量）。

## 面试话术
> "我代码检索用 ripgrep（词法），因为代码要精确符号匹配。向量检索适合自然语言、措辞不对齐的场景，比如按语义搜文档；那需要 embedding 模型+向量库+top-k，生产里常和词法混合。"
