# L4 · 上下文管理

> [!abstract] 核心
> 连续对话让 messages 只增不减，读过的每个文件全文都永久驻留、每圈重发。三道防线：**截断工具输出、压缩历史、守护消息配对。**

## 三道防线

**① 截断工具输出**（`MAX_TOOL_OUTPUT`）：模型不知道你的文件有 5MB，它会兴高采烈 read_file 然后上下文爆炸。
> [!tip] 工具往上下文里灌多少内容，是你的责任，不是模型的。每个可能返回大量数据的工具都要自带上限。

**② 压缩历史**（`compact`）：超阈值时把整段历史丢给模型总结成一段摘要，再替换掉原文。这就是 Claude Code 的 `/compact`。代价：用信息保真度换上下文空间。

**③ 守护配对时机**：压缩只能在一次完整请求**结束后**做，绝不能在 tool_calls 发出、结果还没回时动手。且 `compact` 要**整段替换**而非删几条。详见 [[消息配对约束]]。

## 一个设计手法

把"cwd 已经在工作目录"这个**事实**写进 `run_bash` 的 **description**，而不是 system prompt——关于某个工具怎么用的知识，就该待在那个工具的说明书里。（不过 L6 证明这条软约束仍会被无视。）

## 亲历的坑

- **"不 /compact 再问就报错"**：[[消息配对约束]]被破坏的典型。加了 `sanitize()` 守卫根治。
- **模型瞎编报错原因**："连接池/会话悬挂"全是假的——它看不见自己的源码，见 [[幻觉 模型看不见就编]]。
- **嵌套目录**：模型加 `playground/` 前缀 + write_file 自动建父目录 = 造出 `playground/playground/`。见 [[路径困惑 一个反复出现的坑]]。
