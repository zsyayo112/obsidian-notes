# 🔌 框架无关（Repository Adapter）

> [!important] 一句话
> **核心 agent 不在乎目标仓库用什么技术栈。所有"技术栈知识"关进一个文件 `adapters.py`；核心只从它拿两样——`kind`（只 log 不 branch）和 `test_cmd`（当不透明字符串照跑）。**

## 契约：RepoProfile
核心与技术栈之间**整条边界**就是一个 3 字段小结构体：
```python
@dataclass
class RepoProfile:
    kind: str           # python / rust / go / ... —— 核心只记录，从不据此分叉
    test_cmd: str | None  # 跑测试的命令 —— 核心当不透明字符串照跑，从不解析
    notes: list[str]
```
边界越窄，"框架无关"越硬。

## 机制：detector 注册表
每种技术栈一个小函数，登记进一个列表，按优先级依次问，第一个认领的胜出：
```python
def _detect_rust(root): return RepoProfile("rust","cargo test",...) if (root/"Cargo.toml").exists() else None
_DETECTORS = [_detect_python, _detect_rust, _detect_go, ..., _detect_make]

def detect(root, override=None):
    if override: return RepoProfile("custom", override, ...)   # 逃生门：--test-cmd 一票覆盖
    for d in _DETECTORS:
        if (p := d(root)): return p
    return RepoProfile("unknown", None, ...)                    # 优雅认输，不瞎猜
```
三个设计点：**①顺序即优先级**（语言专属在前、Makefile 兜底垫底，见[[Makefile 是命令别名本]]）**②逃生门**（override 短路一切）**③优雅认输**（认不出返回 None，让 orchestrator 停下问人）。加一种语言 = 写一个 `_detect_xxx` + 登记，核心一行不动。

## 判据：kind 只能被 log，绝不能被 branch
"框架无关"是真是假不靠嘴说，靠 grep 自证：
```bash
grep -rn "\.kind\|kind ==" --include=*.py . | grep -v adapters.py
# 只有一行：orchestrator 把 kind 写进 trace（log），没有任何 if kind == "python"
```
**核心从不因"它是什么项目"改变行为**。哪天有人写下 `if profile.kind == "python":`，框架无关就在那一刻破功——这条 grep 当场抓到它。

## 支持的栈
python / rust / go / java-maven / java-gradle / ruby / node / nestjs / make 兜底 / `--test-cmd` 任意。

## 面试
"你怎么让 agent 核心与技术栈解耦" —— RepoProfile 契约 + detector 注册表 + kind 只 log 不 branch，可现场 grep 自证。**先 Python、NestJS 作第二 adapter** 就是为了实地证明这个卖点。
