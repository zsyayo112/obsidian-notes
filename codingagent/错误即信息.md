# 🔁 错误即信息

> [!important] 一句话
> **对 agent 来说，错误不是故障，是信息。工具失败了不要让程序崩，把错误信息当成"结果"喂回给模型——它会自己读懂、纠正、重试。**

## 原则

工具函数**永远返回字符串，永远不抛异常**：

```python
def call_tool(name, arguments_json):
    try:
        return str(fn(**args))
    except Exception as e:
        return f"错误：{type(e).__name__}: {e}"   # 错误也是一条工具结果
```

## 为什么这样做

- 文件不存在 → 返回 `错误：FileNotFoundError...` → 模型下一圈自己换路径或先 `list_files`。
- 命令执行失败 → 返回 `exit_code=2 ...` → 模型读懂报错，改代码，重跑。
- 参数不是合法 JSON（模型生成的 JSON 偶尔真有语法错）→ 返回错误串而非崩溃。

新手最容易犯的错：让工具异常向上冒泡、把 agent 搞崩。**你把错误藏起来，模型永远学不会；你把错误告诉它，它下一圈就自己修好了。**
