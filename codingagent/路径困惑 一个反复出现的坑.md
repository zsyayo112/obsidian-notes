# 🧭 路径困惑：一个反复出现的坑

> [!bug] 现象
> 模型反复搞不清"自己站在哪个目录"：一会儿 `cd playground`（其实已经在里面），一会儿给路径加 `playground/` 前缀（造出嵌套目录）。同一个坑在五节课里出现了**五次**。

## 四次现形

| 课 | 现象 |
|----|------|
| [[L3 手脚与沙盒]] | `cd playground && python x.py` → `can't cd to playground`（它已经在里面） |
| [[L4 上下文管理]] | `write_file("playground/hello.py")` → 造出嵌套的 `playground/playground/` |
| [[L5 权限与硬约束]] | 用户自己 `find playground` 也踩了同一个坑 |
| [[L6 打磨成品]] | `cd /playground`（连绝对路径都编出来了），且 system prompt 明令禁止也没用 |
| [[L7 子agent与上下文隔离]] | 子 agent 工具锁在 playground 沙盒，却被要求查项目根 → 工具世界 ≠ 任务世界 |

## 根因

**不是模型笨，是"提示词描述的世界"和"工具实际所处的世界"对不齐。**
system prompt 说"你的工作目录是 playground/"，模型合理推断"那我得进去/加前缀"；而工具的 `cwd` 早已锁死在 playground、`_resolve` 又把路径拼在 playground 之上。模型信了前者，工具执行了后者，裂缝就长成真实的错误目录。

> [!tip] 调试第一法则
> 模型行为诡异时，**先假设是你给它的世界模型错了**，而不是它笨。它只能依据你告诉它的信息和工具返回的文本推断现实。你说错一个字，它就会认真地朝错误方向努力——一直到硬盘上多出一个目录。

## 两种修法

- **治标（软约束）**：改 system prompt / 工具 description 说清楚。→ 时灵时不灵，L6 证明它会被无视。
- **治本（硬约束）**：在 `_resolve()` 里无条件剥掉前缀。模型加不加都不影响结果。见 [[硬约束vs软约束]]。

```python
while path.startswith(f"{PLAYGROUND.name}/"):
    path = path[len(PLAYGROUND.name) + 1:]
```
